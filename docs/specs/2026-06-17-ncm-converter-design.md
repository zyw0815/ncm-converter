# NCM 转换器 — 设计文档

- 日期：2026-06-17
- 状态：已通过设计评审，待写实现计划
- 定位：面向个人用户的本地音乐格式转换工具（按真实软件开发流程使用 Git/GitHub）

## 1. 目标与背景

把网易云音乐下载的 `.ncm` 文件转换为通用播放器可直接播放的音频文件，同时完整保留标题、歌手、专辑、封面等元数据。跨平台支持 macOS 与 Windows，提供简约且美观的 GUI。

### 核心原则：保留原始格式优先

NCM 是一层**结构固定**的混淆容器，内部封装的是下载时服务端下发的原始音频流（mp3 或 flac），转换过程**不重新编码、不改变音质**。因此：

- 内部是 flac → 输出 flac
- 内部是 mp3 → 输出 mp3
- 内部是特殊封装（如全景声/杜比，可能为 MP4/EC-3 等）→ 按真实后缀**原样导出**，并在界面标注「特殊格式」，绝不强转、绝不静默产出坏文件

> 关键认知：NCM 容器对所有音质（无损 / 超清母带 / AI 升频 / 全景声）都是统一结构，解密逻辑与内部音频编码无关。劣质工具失败的原因通常是「硬编码输出后缀」或「不读元数据真实格式字段」——本项目通过「先读 JSON format 字段再决定输出」来规避。

## 2. NCM 文件结构与解密流程

```
[magic header: CTENFDAM] [RC4 密钥块(AES 加密)] [元数据 JSON(AES 加密)] [封面] [音频流(RC4 异或)]
```

单文件转换流程：

```
读文件头 CTENFDAM ──否──→ 标记「非 NCM，跳过」
   │是
AES-128-ECB 解出 RC4 密钥 → 解出元数据 JSON → 提取封面
   │
读 JSON 的 format 字段
   ├─ flac → 还原音频流，写 .flac，写入标签 + 封面
   ├─ mp3  → 还原音频流，写 .mp3，写入标签 + 封面
   └─ 其他 → 按真实后缀原样导出，标注「特殊格式」
   │
(若开启「强制转 WAV/FLAC」) → 调 ffmpeg 转码 → 重新写入标签
```

- AES 使用 `pycryptodome`，RC4 keystream 自实现，全程离线、无需联网。
- 已是 flac（非 ncm）的文件：直接复制并补全标签，不做无意义的解密尝试。

## 3. 架构（核心与界面分离）

```
ncm-converter/
├─ core/                  纯逻辑，不依赖 GUI，可独立单元测试
│   ├─ ncm.py             解密单个 NCM：解壳 → 出密钥 → 读元数据 → 还原音频
│   ├─ metadata.py        用 mutagen 把 标题/歌手/专辑/封面 写回 flac/mp3
│   ├─ formats.py         读 format 字段，判定真实格式（flac/mp3/特殊）
│   ├─ transcode.py       可选：调 ffmpeg 转 WAV/FLAC
│   └─ converter.py       编排单文件流程，返回结构化结果
├─ gui/                   PyQt6 界面，仅负责展示与调度
│   ├─ main_window.py
│   ├─ task_model.py      队列数据模型（每行一首歌的状态）
│   └─ workers.py         QThreadPool 多线程，界面不卡顿
├─ tests/
└─ main.py
```

**分层理由**：`core/` 完全不碰界面，纯输入输出，可用真实 NCM 样本写单元测试；界面与逻辑互不牵连，单个文件职责单一、便于维护。

### 单文件结果对象

`converter.py` 对每个文件返回：

```
{ 源路径, 状态, 识别到的标题/歌手/专辑/真实格式, 输出路径, 失败原因 }
```

界面据此渲染列表行与失败清单。

## 5. 功能清单

### 基础必备（默认全部包含）

- 拖拽导入文件或文件夹
- 递归扫描子文件夹，自动找出所有 `.ncm`
- 批量队列：每首歌单独进度 + 整体进度
- 多线程并行转换（QThreadPool，界面不卡）
- 转换前预览：列表直接显示 标题 / 歌手 / 专辑 / 真实格式 / 封面缩略图
- 智能跳过：自动识别并跳过 非 ncm、损坏文件，不中断整批
- 已是 flac 的文件直接复制并补全标签
- 失败清单 + 一键重试失败项，失败原因人话化
- 冲突处理：输出已存在时 跳过 / 覆盖 / 自动重命名
- 完成后一键打开输出目录

### 可选功能（用户已选定全部启用）

- 输出文件名模板（默认 `{歌手} - {标题}`，可选 `{专辑}/{标题}` 等）
- 保留源目录结构（选文件夹批量时，在输出目录复刻子文件夹层级）
- 深色 / 浅色主题切换
- 转换后删除原 NCM（默认关闭；仅在确认转换成功后删除；首次操作弹一次二次确认）
- 可选「强制转 WAV/FLAC」开关（默认关闭；开启时通过 ffmpeg 转码）

## 6. 错误处理

- 单个文件失败**绝不中断整批**，仅标红该行并写明原因。
- 失败原因人话化：`非 NCM 文件` / `文件损坏` / `全景声等特殊格式，已原样导出` / `输出目录无写入权限` / `ffmpeg 未找到`。
- 删除原文件：仅在转换确认成功后执行，且首次操作弹一次确认框。

## 7. 依赖与打包

- **PyQt6**（界面）、**mutagen**（标签读写）、**pycryptodome**（AES）。
- **ffmpeg**：仅「强制转码」时调用；优先检测系统已安装，未安装时给出明确提示（或随包附带）。
- 解密为纯 Python，离线运行。
- 打包：PyInstaller 分别产出 macOS `.app` 与 Windows `.exe`。

## 8. 测试

`core/` 用真实 NCM 样本编写单元测试，覆盖四种情况：

1. flac-NCM → 输出 flac，标签与封面正确
2. mp3-NCM → 输出 mp3，标签与封面正确
3. 非 NCM 文件 → 正确识别并跳过
4. 损坏文件 → 不崩溃，返回明确失败原因

> 测试样本为受版权音频，仅本地使用，已在 `.gitignore` 中排除，不入仓库。

## 9. 范围之外（YAGNI）

- 不支持其他播放器的加密格式（如 QQ 音乐 qmc / 酷狗 kgm）。
- 不做在线音乐下载、不联网。
- 不做音质「提升」/AI 升频（NCM 内已是何种音质即输出何种）。
